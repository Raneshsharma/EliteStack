## 1. Overview

P0.3 adds resume management capabilities on top of the CRUD built in P0.2. The primary new feature is **duplicate resume** ÔÇö copying an existing resume as a starting point for a tailored version. Most other CRUD operations (list, view, edit, delete, set primary) were built as part of P0.2's builder page and supporting views. P0.3 tightens those up and adds a 403 page for cross-user access attempts.

**Goal:** A logged-in user can perform all CRUD operations on their resumes without error. They can identify their default resume at a glance and duplicate any resume in one click.

## 3. What's New

### 3.1 Duplicate Resume Feature

**Model Change:**
- Add `copied_from` nullable FK to `Resume` model pointing to `self`
- `copied_from = models.ForeignKey('self', on_delete=models.SET_NULL, null=True, blank=True, related_name='copies')`

**New View: `resume_duplicate`**
- Route: `POST /resumes/<int:resume_id>/duplicate/`
- Logic:
  1. Load source resume with all related entries (`select_related` + `prefetch_related`)
  2. Create new `Resume` with `title = f"Copy of {source.title}"`, `is_primary=False`, `status='draft'`, `copied_from=source`
  3. Iterate over source's education, experience, projects, skills and create copies pointing to the new resume
  4. Save all in a single `transaction.atomic` block
  5. Return redirect to `resume_list` with success message
- Permission: same user-scoping as all other views ÔÇö if resume doesn't belong to user, `get_object_or_404` raises 404 (not 403, since we don't reveal existence)

**Template: `resume_delete.html` (reuse for duplicate confirmation)**
- A simple confirmation page for delete already exists
- Duplicate does NOT need a confirmation page per spec ÔÇö "Confirm Ôćĺ POST Ôćĺ redirected"

**Template: `resume_list.html` update**
- Add "Duplicate" button to each card (between "Edit" and "Delete")
- Icon: clipboard-duplicate icon (Heroicons `document-duplicate`)
- Form: `<form action="{% url 'resume_duplicate' resume.id %}" method="post">` with CSRF token
- Button style: ghost/secondary, like the other action buttons

**URL:**
- `path('resumes/<int:resume_id>/duplicate/', views.resume_duplicate, name='resume_duplicate')`

**View:**
```python
@login_required
def resume_duplicate(request, resume_id):
    """Duplicate an existing resume."""
    source = get_object_or_404(Resume, id=resume_id, user=request.user)
    with transaction.atomic():
        new_resume = Resume.objects.create(
            user=request.user,
            title=f"Copy of {source.title}",
            is_primary=False,
            status='draft',
            copied_from=source,
        )
        for edu in source.education.all():
            edu.pk = None
            edu.resume = new_resume
            edu.save()
        for exp in source.experience.all():
            exp.pk = None
            exp.resume = new_resume
            exp.save()
        for proj in source.projects.all():
            proj.pk = None
            proj.resume = new_resume
            proj.save()
        for skill in source.skills.all():
            skill.pk = None
            skill.resume = new_resume
            skill.save()
    messages.success(request, f'"{source.title}" duplicated successfully.')
    return redirect('resume_list')
```

**Tests:**
- `test_duplicate_creates_new_resume`: POST duplicate Ôćĺ new resume exists with correct title
- `test_duplicate_copies_all_sections`: Source with education/experience/projects/skills Ôćĺ all copied
- `test_duplicate_sets_copied_from`: New resume's `copied_from` equals source
- `test_duplicate_sets_draft_status`: Duplicated resume is always draft regardless of source status
- `test_duplicate_user_isolation`: User cannot duplicate another user's resume (404)

### 3.2 403 Forbidden Page

**Template: `403.html`** (at `templates/403.html`)
- Extends `base.html`
- Centered content: lock icon, "403 ÔÇö Access Forbidden", message: "You don't have permission to access this page."
- Button: "Back to Dashboard" Ôćĺ `/resumes/`
- HTTP status: return `HttpResponseForbidden` (or use `render()` with 403 status)

**Update `resume_update` and `resume_preview` views:**
- Change from `get_object_or_404` to manual fetch + 403 check:
```python
resume = get_object_or_404(Resume, id=resume_id)
if resume.user != request.user:
    return render(request, '403.html', status=403)
```
- Or use a helper: a decorator `@owner_required(model=Resume, id_param='resume_id')`

**Simple helper approach (preferred):**
```python
def _get_user_resume(request, resume_id):
    resume = get_object_or_404(Resume, id=resume_id)
    if resume.user != request.user:
        raise Http404("Resume not found")
    return resume
```
This returns 404 rather than 403 ÔÇö standard Django behavior. For a proper 403, add the 403 template and use `PermissionDenied`.

**Spec says 403.** Use `PermissionDenied` exception:
```python
from django.core.exceptions import PermissionDenied

def _get_user_resume(request, resume_id):
    resume = get_object_or_404(Resume, id=resume_id)
    if resume.user != request.user:
        raise PermissionDenied
    return resume
```
