> **Announced at:** 2026-04-19
> **Epic:** P0.5 Resume Export (from `docs/superpowers/specs/P0.md`)

---

## 1. Overview

P0.5 adds a real PDF download feature using reportlab server-side generation. The endpoint reads the resume's `template_style` to determine which layout approach to use (Classic = serif, Modern = sans-serif with blue accents, Minimal = clean serif).

**Approach:** reportlab draws the resume programmatically onto a PDF canvas. This is a programmatic layout (not HTML-to-PDF), so the layout is defined in Python code. The export view handles page size (Letter), margins (0.5in), fonts, section spacing, and page breaks.

### 3.2 Export View (reportlab)

**File:** `d:/EliteSttack/resumes/views.py`

reportlab draws the resume programmatically on a PDF canvas. The approach:

1. Create a `BytesIO` buffer and `canvas.Canvas(buffer, pagesize=letter)`
2. Track vertical position `y` (starts at top margin)
3. Draw header: name, contact line
4. For each section (Experience, Education, Projects, Skills): draw section title, then entries
5. When `y` drops below bottom margin, call `canvas.showPage()` and reset `y`
6. On final entry, `canvas.save()` and return buffer contents as `HttpResponse`

**Key reportlab imports:**
```python
from io import BytesIO
from reportlab.lib.pagesizes import letter
from reportlab.lib.units import inch
from reportlab.pdfgen import canvas
```

**Page setup:** Letter (612 × 792 pts), 0.5in margins (36pt each side), left margin at 36pt, right margin at 576pt, content width = 540pt.

**Filename:** Sanitize `full_name` or `title` to ASCII, replace spaces with `_`, append `_Resume.pdf`.

**URL:** `path('resumes/<int:resume_id>/export/', views.resume_export_pdf, name='resume_export_pdf')`

### 3.3 Export PDF Button (View-Only Page)

**File:** `d:/EliteSttack/templates/resumes/resume_preview.html`

The current "Print / Export" button uses `window.print()`. Replace it with a proper PDF download button.

Find the actions bar (lines 7-23). Replace the entire actions bar with:

```html
    <!-- Actions Bar -->
    <div class="flex justify-between items-center mb-4 print:hidden">
        <h1 class="text-xl font-bold text-gray-900">{{ resume.title }}</h1>
        <div class="flex gap-2">
            <button
                id="export-pdf-btn"
                data-resume-id="{{ resume.id }}"
                data-status="{{ resume.status }}"
                class="px-4 py-2 bg-blue-600 text-white rounded-md hover:bg-blue-700 flex items-center gap-2 text-sm"
            >
                <svg class="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                    <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M4 16v1a3 3 0 003 3h10a3 3 0 003-3v-1m-4-4l-4 4m0 0l-4-4m4 4V4"></path>
                </svg>
                Export PDF
            </button>
            <button onclick="window.print()" class="px-4 py-2 border border-gray-300 rounded-md text-gray-700 hover:bg-gray-50 text-sm">
                Print
            </button>
            <a href="{% url 'resume_update' resume.id %}" class="px-4 py-2 border border-gray-300 rounded-md text-gray-700 hover:bg-gray-50 text-sm">
                Edit
            </a>
            <a href="{% url 'resume_list' %}" class="px-4 py-2 border border-gray-300 rounded-md text-gray-700 hover:bg-gray-50 text-sm">
                Back
            </a>
        </div>
    </div>
```

Add JavaScript at the bottom of the template (before `</body>` or in a `<script>` block at end of `{% block content %}`):

```html
<script>
(function() {
    var btn = document.getElementById('export-pdf-btn');
    if (!btn) return;
    btn.addEventListener('click', function() {
        var resumeId = btn.dataset.resumeId;
        var status = btn.dataset.status;
        if (status === 'draft') {
            if (!confirm('This resume is incomplete (draft). Export anyway?')) return;
        }
        btn.disabled = true;
        btn.innerHTML = '<svg class="w-4 h-4 animate-spin" fill="none" stroke="currentColor" viewBox="0 0 24 24"><circle class="opacity-25" cx="12" cy="12" r="10" stroke="currentColor" stroke-width="4"></circle><path class="opacity-75" fill="currentColor" d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4zm2 5.291A7.962 7.962 0 014 12H0c0 3.042 1.135 5.824 3 7.938l3-2.647z"></path></svg> Generating...';
        var form = document.createElement('form');
        form.method = 'POST';
        form.action = '/resumes/' + resumeId + '/export/';
        var csrfInput = document.createElement('input');
        csrfInput.type = 'hidden';
        csrfInput.name = 'csrfmiddlewaretoken';
        csrfInput.value = '{{ csrf_token }}';
        form.appendChild(csrfInput);
        document.body.appendChild(form);
        form.submit();
        setTimeout(function() {
            btn.disabled = false;
            btn.innerHTML = '<svg class="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M4 16v1a3 3 0 003 3h10a3 3 0 003-3v-1m-4-4l-4 4m0 0l-4-4m4 4V4"></path></svg> Export PDF';
        }, 3000);
    });
})();
</script>
```

The JS submits a POST to `/resumes/<id>/export/` (WeasyPrint generation), which returns the PDF as a download.

### 3.6 Template Parity Note

reportlab generates PDFs programmatically — it does not use the HTML templates. The PDF layout is defined in Python code and may differ slightly from the web preview. This is acceptable for MVP. Future work could add WeasyPrint (with GTK3 available) for exact HTML-to-PDF parity.

## 6. Out of Scope (per P0.md)

- DOCX / Word export
- PNG / image export
- Batch export (multiple resumes at once)
- Custom page margins per resume
- Watermark on exported PDFs
- Email-to-recruiter directly from app
- Direct upload to job portals
- Custom branding / logo in footer
- Password-protected PDFs
