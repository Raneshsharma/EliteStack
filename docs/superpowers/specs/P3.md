## P3.1: AI Resume Rewrite

**Primary Owner:** `Frontend Tech Lead`, `Backend Tech Lead`
**Support:** `AI Integration Agent`, `Content Rewrite Agent`, `Form/Validation Agent`, `API Agent`, `UI Component Agent`, `Edge Case Agent`, `QA Lead`, `Functional Test Agent`
**Watchers:** `Product Manager Agent`, `System Architect`, `Project Manager Agent`

### 1. Problem Statement

**Who has this problem?** Every job seeker who has written resume descriptions but suspects they could be more impactful — lacking strong action verbs, specific metrics, or professional language.

**What is the problem?** Most users, especially early-career job seekers, write flat, passive descriptions like "Responsible for answering emails" instead of "Managed inbound client communications, reducing response time by 40%." Without guidance, resumes fail to pass ATS scanners and don't impress recruiters.

**Why does it matter?** The difference between a mediocre resume and a compelling one can mean the difference between 5 interviews and 0. AI-powered rewriting gives users instant access to professional-level writing without hiring a resume writer ($100–$500).

**Current workaround (if any):** Users hire professional resume writers, use generic templates, or manually revise with limited knowledge of what makes bullet points effective.

---

### 2. Goal / Objective

**User Goal:** Rewrite a single bullet point or full description section with AI-powered suggestions that are more specific, action-oriented, and ATS-friendly.

**Business Goal:** Differentiate the product from non-AI resume builders. Drive Pro/Premium subscription conversions. Build trust in AI quality by making rewrites feel like they came from a professional writer.

**Success Definition:** Users can select any description text, click "Rewrite with AI," and receive 3 alternative rewrites within 5 seconds. They can accept, regenerate, or edit any suggestion. The rewrite is measurably better (more specific, action-oriented) than the original.

---

### 3. Scope

**In-Scope:**
- **Text selection:** User selects a bullet point or full description field in the builder
- **Rewrite trigger:** "Rewrite with AI" button appears when text is selected
- **AI rewrite generation:** Generate 3 alternative rewrites per request using GPT-4
- **Rewrite display:** Show 3 alternatives in a modal with "Accept", "Regenerate", "Edit" options
- **Accept rewrite:** Replace the original text with the accepted rewrite
- **Regenerate:** Generate 3 new alternatives (uses an additional AI call)
- **Edit:** Open accepted rewrite in a text editor before applying
- **Character limit:** Rewrite requests limited to 500 words of input per call
- **Usage tracking:** Track number of rewrites per user for analytics and rate limiting
- **Pro tier gate:** Feature available on Pro and Premium tiers only

**Out-of-Scope:**
- Bulk rewrite (rewrite all bullets at once)
- AI scoring of rewrite quality
- Rewrite history / undo
- Custom tone settings (formal, casual, etc.)
- Industry-specific rewrite styles
- Rewrite in languages other than English

---

### 4. User Stories

**US-P3.1-A:** As a **logged-in user on Pro/Premium**, I want to **select text in my resume description** and click "Rewrite with AI", so that I can **get stronger, more professional rewrites**.

**US-P3.1-B:** As a **logged-in user on Pro/Premium**, I want to **see 3 alternative rewrites** side by side, so that I can **choose the one that best fits my experience**.

**US-P3.1-C:** As a **logged-in user on Pro/Premium**, I want to **regenerate alternatives** if none of the 3 options suit me, so that I can **get better options**.

**US-P3.1-D:** As a **logged-in user on Pro/Premium**, I want to **edit the AI rewrite before accepting it**, so that I can **fine-tune the language to my liking**.

**US-P3.1-E:** As a **Free tier user**, I want to **see a clear upgrade prompt** when I try to use AI features, so that I understand **what's available on my tier**.

---

### 5. User Flow / Experience

**Rewrite Flow:**
1. User selects text in a description field (Experience or Projects)
2. A floating toolbar appears above the selection with "Rewrite with AI" button
3. User clicks "Rewrite with AI"
4. Loading state: "Generating rewrites..." with spinner (up to 5 seconds)
5. Modal opens showing 3 rewrite alternatives, each in a card:
   - Original text shown at the top (collapsed, expandable)
   - 3 rewritten alternatives below, each with: text, "Use This" button, "Edit" button
6. User clicks "Use This" on one alternative
7. Text is replaced in the form field
8. Toast: "Resume updated"

**Regenerate Flow:**
1. User clicks "Regenerate" below the alternatives
2. Loading state appears
3. New 3 alternatives replace the current ones

**Edit Flow:**
1. User clicks "Edit" on a rewrite card
2. Text area becomes editable inline
3. User modifies the text
4. User clicks "Use This" to apply

**Free Tier Gate Flow:**
1. Free tier user selects text
2. Floating toolbar shows "Rewrite with AI" button
3. User clicks the button
4. Upgrade modal appears: "This feature is available on Pro and Premium plans. Upgrade to unlock AI rewrites."
5. "See Plans" button → navigates to pricing section on landing page
6. "Maybe Later" dismisses the modal

### 7. Edge Cases & Error Handling

| Scenario | Expected Behavior |
|----------|-------------------|
| AI API returns an error (timeout, 500) | Show error toast: "AI rewrite failed. Please try again." + log error |
| AI returns fewer than 3 alternatives | Display what was returned with a note. Regenerate available. |
| AI returns an empty rewrite | Discard empty results. Show a warning: "Could not generate a rewrite for this text." |
| User has no text selected | "Rewrite with AI" button is hidden or disabled |
| Text selected is too long (> 500 words) | Show warning before making API call. Do not call the API. |
| User is on free tier | Show upgrade modal. Do not call the API. |
| Rate limit reached (too many rewrites) | Show: "You've used all your AI rewrites for now. Upgrade for more." |

### 9. Dependencies & Assumptions

**Dependencies:**
- OpenAI GPT-4 API access
- P0.2 (Resume Builder) — rewrite is embedded in the builder

- Rate limiting infrastructure (Redis or in-memory)

**Assumptions:**
- GPT-4 produces sufficiently professional rewrites without extensive prompt engineering
- 3 alternatives is the right number — not too few, not overwhelming
- Text selection via browser APIs is reliable across browsers
- A 5-second SLA is achievable for GPT-4 API calls with proper caching

**Blockers if missing:**
- OpenAI API credentials must be configured
- User subscription tier system must be implemented

---

### 10. Technical Notes

- **API Endpoint:** `POST /api/resume/<id>/rewrite/` — accepts `{section_type, field_name, selected_text}`. Returns `{alternatives: [string, string, string]}`.
- **OpenAI Integration:** Use `openai` Python SDK. System prompt instructs the model to produce 3 professional rewrites with action verbs, metrics, and impact statements.
- **Prompt Engineering:** System prompt: "You are a professional resume writer. Rewrite the following resume bullet point to be more specific, action-oriented, and ATS-friendly. Return exactly 3 alternatives, each on a separate line starting with '1.', '2.', '3.'."
- **Rate Limiting:** Track rewrite count per user in Redis. Limit: 20 rewrites/day for Pro, 50/day for Premium. Reset at midnight UTC.
- **Safety:** Add server-side content filter on AI output. Block responses containing PII, harmful instructions, or off-topic content.
- **Frontend:** Use JavaScript `window.getSelection()` to detect text selection. Show floating toolbar as a positioned `<div>`. Use `fetch()` to call the rewrite API.
- **Caching:** Do NOT cache rewrite results — each rewrite request is unique to the user's input text.
- **Error Handling:** Catch `openai.APIError`. Return appropriate HTTP status codes. Log all errors with user_id and timestamp.

## P3.2: Job Match Rewrite

**Primary Owner:** `Frontend Tech Lead`, `Backend Tech Lead`
**Support:** `AI Integration Agent`, `Job Match Agent`, `Content Rewrite Agent`, `Form/Validation Agent`, `UI Component Agent`, `QA Lead`, `Functional Test Agent`
**Watchers:** `Product Manager Agent`, `System Architect`, `Project Manager Agent`

### 1. Problem Statement

**Who has this problem?** Every job seeker who applies to a specific role and wants their resume to align with the job description — but doesn't know which keywords to include, which skills to emphasize, or how to reframe their experience.

**What is the problem?** Most job seekers submit the same resume to every job. ATS scanners and recruiters immediately reject resumes that don't match the job description's keywords. A generic resume sent to a specific job posting is significantly less effective than one tailored to the role.

**Why does it matter?** Tailoring a resume to each job posting can increase interview callback rates by 40–60%. Most users don't have the time or knowledge to manually optimize each resume for each job. AI-powered job matching bridges this gap instantly.

**Current workaround (if any):** Users manually copy-paste job descriptions into their resume, look for keywords manually, or hire resume writers for $200–$500 per job application.

---

### 2. Goal / Objective

**User Goal:** Paste a job description, and have AI automatically rewrite my resume bullets to match the keywords, skills, and tone of the job posting — in under 30 seconds.

**Business Goal:** Drive Pro/Premium upsells with a high-value feature. Position the product as an AI-powered job application tool, not just a resume builder. Increase interview callback rates as reported by users.

**Success Definition:** User can paste a job description, and within 30 seconds, receive a rewritten resume that includes the most relevant keywords from the job description while maintaining factual accuracy.

---

### 3. Scope

**In-Scope:**
- **Job description input:** Large textarea for pasting job posting text (up to 5,000 characters)
- **AI matching and rewrite:** Analyze job description for keywords and skills, rewrite relevant resume sections to match
- **Keyword highlighting:** Highlight which keywords were added/referenced in the rewrite
- **Match score:** Show a match percentage (e.g., "Your resume is now 85% aligned with this job")
- **Section selection:** User selects which sections to rewrite (Experience, Skills, Summary)
- **ATS preview:** Show which ATS keywords from the job posting are now present in the resume
- **Save options:** Save rewritten content to the current resume or export as a new resume
- **Usage tracking:** Track job match usage per user
- **Pro tier gate:** Available on Pro and Premium tiers

**Out-of-Scope:**
- Auto-scraping job postings from URLs (future)
- Multi-job tracking (match to multiple jobs simultaneously)
- Cover letter generation (P3.3)
- LinkedIn job posting integration (future)
- Salary-based job matching
- Industry-specific resume templates

---

### 4. User Stories

**US-P3.2-A:** As a **logged-in user on Pro/Premium**, I want to **paste a job description** into a text area, so that I can **prepare my resume for that specific role**.

**US-P3.2-B:** As a **logged-in user on Pro/Premium**, I want to **see which sections of my resume will be rewritten**, so that I can **choose what to tailor**.

**US-P3.2-C:** As a **logged-in user on Pro/Premium**, I want to **see a match score** after rewriting, so that I can **know how well my resume aligns with the job**.

**US-P3.2-D:** As a **logged-in user on Pro/Premium**, I want to **see which keywords were added**, so that I can **understand what changed and learn for future applications**.

**US-P3.2-E:** As a **logged-in user on Pro/Premium**, I want to **save the rewritten resume as a new resume** or apply it to my current resume, so that I can **keep both versions**.

---

### 9. Dependencies & Assumptions

**Dependencies:**
- OpenAI GPT-4 API (function calling or structured output)
- P0.2 (Resume Builder) — job match rewrites builder content
- P0.3 (Resume Management) — "Save as New Resume" creates a new resume
- User subscription tier detection

**Assumptions:**
- GPT-4 can extract relevant keywords from job descriptions accurately
- Keyword matching can be done via string similarity (no specialized ATS parser needed)
- Users will accept most AI rewrites — minimal need for manual editing post-match
- Job descriptions up to 5,000 words provide sufficient context for accurate matching
- Most users will paste job descriptions from LinkedIn or Indeed

---

### 10. Technical Notes

- **API Endpoint:** `POST /api/resume/<id>/job-match/` — accepts `{job_description, sections_to_rewrite: [str], resume_id}`. Returns `{match_score: int, rewritten_sections: {section: text}, keywords_found: [str], keywords_matched: [str]}`.
- **Keyword Extraction:** Use GPT-4 to extract top 15 keywords/skills from job description. Then compare against resume content using simple TF-IDF or keyword overlap scoring.
- **Match Score Calculation:** `match_score = (matched_keywords / total_job_keywords) * 100`. Round to nearest integer.
- **Rewrite Prompt:** "Analyze the following job description and resume. Rewrite the selected sections to incorporate relevant keywords and phrasing from the job description while maintaining factual accuracy. Do not invent skills or experiences not present in the original resume."
- **Keyword Highlighting:** Frontend uses `<mark>` tags to highlight matched keywords in the rewritten text preview.
- **Privacy:** Do not store job descriptions in the database. Process and discard immediately. Log only anonymized usage metrics.
- **Rate Limiting:** 5 job matches per day for Pro, 20 per day for Premium.

## P3.3: Cover Letter Generator

**Primary Owner:** `Frontend Tech Lead`, `Backend Tech Lead`
**Support:** `AI Integration Agent`, `Content Generation Agent`, `Form/Validation Agent`, `UI Component Agent`, `QA Lead`, `Functional Test Agent`
**Watchers:** `Product Manager Agent`, `System Architect`, `Project Manager Agent`

### 1. Problem Statement

**Who has this problem?** Every job seeker who needs to write a cover letter for a job application — which is required by most professional job postings.

**What is the problem?** Writing a cover letter from scratch is time-consuming and difficult. Most job seekers either skip them (losing out on applications that require them), use generic templates (which don't impress recruiters), or spend 30–60 minutes writing one that still feels weak.

**Why does it matter?** Cover letters are a key differentiator — a strong cover letter can dramatically increase the chances of an interview by demonstrating genuine interest, communication skills, and cultural fit. Most competitors don't offer AI cover letter generation, making this a strong differentiation point.

**Current workaround (if any):** Users write cover letters manually in Word, use generic templates, or skip them entirely.

---

### 2. Goal / Objective

**User Goal:** Generate a professional, personalized cover letter for a specific job in under 60 seconds — by providing the job title, company name, and optional job description.

**Business Goal:** Add a high-value, time-saving feature that complements the resume builder. Drive Pro/Premium subscriptions. Keep users engaged on the platform through the full job application process.

**Success Definition:** User inputs a job title and company name, optionally a job description, and receives a complete, professionally formatted cover letter within 60 seconds. The letter references specific details from the job description and the user's resume.

---

### 3. Scope

**In-Scope:**
- **Cover letter input form:** Job title, company name, hiring manager name (optional), job description (optional textarea)
- **AI cover letter generation:** Generate a complete, 300–500 word cover letter using resume data + job details
- **Cover letter display:** Render the generated letter in a clean, professional format
- **Edit capability:** User can edit the generated letter before saving
- **Save options:** Save to the associated resume as an attached document, or download as text
- **Template styles:** Match the style of the selected resume template (Classic, Modern, Minimal)
- **Multiple versions:** User can generate multiple cover letters for the same job application
- **Usage tracking:** Track generation count per user
- **Pro tier gate:** Available on Pro and Premium tiers

**Out-of-Scope:**
- Direct email integration (send cover letter directly from the app)
- Cover letter templates gallery
- Cover letter evaluation/scoring
- Multi-paragraph customization (user selects which paragraphs to regenerate)
- Industry-specific cover letter styles
- Language translation

---

### 4. User Stories

**US-P3.3-D:** As a **logged-in user on Pro/Premium**, I want to **save the cover letter to my resume profile**, so that I can **access it later for future applications**.

## P3.4: ATS Score Checker

**Primary Owner:** `Frontend Tech Lead`, `Backend Tech Lead`
**Support:** `AI Integration Agent`, `ATS Analysis Agent`, `Form/Validation Agent`, `UI Component Agent`, `QA Lead`, `Functional Test Agent`
**Watchers:** `Product Manager Agent`, `System Architect`, `Project Manager Agent`

### 1. Problem Statement

**Who has this problem?** Every job seeker applying to positions at large companies that use Applicant Tracking Systems (ATS) — which is the majority of Fortune 500 companies.

**What is the problem?** Up to 75% of large company applications are rejected by ATS before a human ever sees them. Job seekers have no way to know if their resume will pass the ATS screening — they submit blindly and wonder why they never hear back.

**Why does it matter?** Users who know their ATS score can fix problem areas before applying, dramatically increasing their chances of reaching the human review stage. This feature provides immediate, actionable feedback — one of the most compelling AI features.

**Current workaround (if any):** Users pay $20–$100 for ATS scan services, use free tools with limited functionality, or submit without any ATS feedback.

---

### 2. Goal / Objective

**User Goal:** Upload a job description or paste one in, and instantly see an ATS compatibility score with specific feedback on what to fix — so I can submit with confidence.

**Business Goal:** Drive Pro/Premium subscriptions with a high-visibility, high-value feature. Position the product as the most intelligent resume tool on the market.

**Success Definition:** User pastes a job description and receives an ATS score (0–100) with categorized feedback (keywords, formatting, length, readability) within 10 seconds.

---

### 3. Scope

- **Scoring Algorithm:**
  - Keyword Match: `count(matched_keywords) / count(job_keywords) * 100`
  - Formatting Score: heuristic check (single column +10, proper section headers +10, no tables/images +10, standard fonts +10)
  - Readability Score: Flesch-Kincaid grade level. Target: 8–12. Score based on distance from target.
  - Length Score: 1 page = 100, 1–2 pages = 80, < 1 page = 60, > 2 pages = 40
- **Keyword Extraction:** Use GPT-4 to extract 15–20 key skills/requirements from job description.

- **Data Source:** Embedded JSON dataset of ~500 common job titles with salary ranges by experience level. Sources: BLS, Glassdoor public data. Updated annually.
- **Cost of Living:** Built-in COL index for 200+ US cities (based on C2ER COLI). Formula: `adjusted_salary = base_salary * (target_coli / base_coli)`.
- **API Endpoint:** `GET /api/salary/calculate/?title=<title>&location=<location>&level=<level>` — returns `{min, median, max, currency, location, title, level}`.
- **Autocomplete:** Frontend uses a pre-loaded JSON index for autocomplete. No API call for autocomplete.
- **No AI Required:** This feature is purely data-driven. No OpenAI API needed. Reduces operational cost.

## P3.9: Portfolio Builder

**Primary Owner:** `Frontend Tech Lead`, `UI Component Agent`
**Support:** `Template Architect Agent`, `CSS/Animation Agent`, `Backend Tech Lead`, `QA Lead`, `Functional Test Agent`
**Watchers:** `Product Manager Agent`, `System Architect`, `Project Manager Agent`

### 1. Problem Statement

**Who has this problem?** Every early-career job seeker who has personal projects, a GitHub profile, design work, or writing samples they want to showcase alongside their resume — but has no web development skills to build a portfolio.

**What is the problem?** Most employers, especially in tech, design, and content roles, want to see actual work samples. A resume alone doesn't demonstrate skills. Building a portfolio website requires coding knowledge or expensive services — creating a barrier for most job seekers.

**Why does it matter?** A portfolio dramatically increases credibility and interview chances for creative and technical roles. By integrating a portfolio builder with the resume tool, the product becomes a one-stop career branding platform — increasing retention and premium subscriptions.

**Current workaround (if any):** Users build free portfolios on Wix, Squarespace, GitHub Pages, or Notion — all of which require significant setup time and don't integrate with their resume.

---

### 2. Goal / Objective

**User Goal:** Select portfolio projects from my resume's project section, customize the layout and theme, and publish a personal portfolio page — without writing code.

**Business Goal:** Expand the product from "resume builder" to "career branding platform." Drive Premium subscriptions with a unique, high-value feature. Increase user engagement by giving them a public URL to share with recruiters.

**Success Definition:** User can create a portfolio page from their resume data in under 5 minutes, customize the theme, and receive a shareable public URL within 10 minutes of starting.

---

### 3. Scope

**In-Scope:**
- **Portfolio page generation:** Auto-populate from resume project section and experience entries
- **Customization options:**
  - Theme selection (3–5 preset color schemes)
  - Layout selection (single column, grid, showcase)
  - Profile photo upload (optional)
  - Bio / "About Me" section (from resume summary)
  - Social links (LinkedIn, GitHub, Twitter)
- **Project showcase:** Each project from the resume as a portfolio card with image, title, description, link
- **Public URL:** Each user gets a unique portfolio URL (e.g., `/u/username/`)
- **Resume link:** Portfolio includes a "Download My Resume" button linking to the user's resume
- **Custom domain (Premium):** Premium users can connect a custom domain
- **Pro/Premium tier gate:** Free users get a limited portfolio (3 projects, watermark); Premium gets unlimited

**Out-of-Scope:**
- Full website builder (multi-page, custom HTML/CSS)
- Blog or CMS integration
- Contact form on portfolio
- Analytics dashboard for portfolio
- SEO optimization for portfolio pages (beyond basic meta tags)

---

### 4. User Stories

**US-P3.9-B:** As a **logged-in user on Premium**, I want to **customize the theme and layout** of my portfolio, so that it **reflects my personal brand**.

**US-P3.9-C:** As a **logged-in user on Premium**, I want a **shareable public URL**, so that I can **send my portfolio to recruiters and hiring managers**.

**US-P3.9-D:** As a **Premium user**, I want to **connect a custom domain** to my portfolio, so that I can **present a professional portfolio URL**.

7. User clicks "Publish Portfolio"
8. Confirmation: "Your portfolio is live at [URL]" with copy link button

## P3.10: AI Chat Interface

**Primary Owner:** `Frontend Tech Lead`, `Backend Tech Lead`
**Support:** `AI Integration Agent`, `UI Component Agent`, `Form/Validation Agent`, `State Management Agent`, `QA Lead`, `Functional Test Agent`
**Watchers:** `Product Manager Agent`, `System Architect`, `Project Manager Agent`

### 1. Problem Statement

**Who has this problem?** Every job seeker who has a specific question during their career journey — about resume writing, job searching, negotiating, networking — and needs an instant, intelligent answer without searching the web.

**What is the problem?** Job seekers have hundreds of specific, context-dependent questions that Google can't answer well ("Should I put my GPA on my resume if it's 3.0 and I'm applying to Google?"). Existing web search results are generic and require users to synthesize information themselves.

**Why does it matter?** An AI chat interface positioned as a "career copilot" creates the deepest form of product engagement. Users who chat regularly become power users, driving high retention and premium conversion. It also differentiates the product as an always-available career advisor, not just a resume tool.

**Current workaround (if any):** Users ask friends, mentors, Reddit communities, or pay for career coaching ($50–$500/hr). Most give up and make uninformed decisions.

---

### 2. Goal / Objective

**User Goal:** Ask any career-related question in natural language and receive an intelligent, context-aware answer — drawing from my resume data when relevant.

**Business Goal:** Create the deepest form of product engagement. Make the AI chat the primary reason users return daily. Differentiate the product as an "always-on career advisor."

**Success Definition:** User can type a career question in natural language and receive a relevant, specific, actionable answer within 5 seconds. Chat maintains conversation context across multiple messages.

---

### 3. Scope

**In-Scope:**
- **Chat interface:** Persistent chat panel accessible from the dashboard and builder
- **Natural language questions:** User can ask any career-related question
- **Context awareness:** Chat has access to the user's resume data (with permission) for personalized answers
- **Conversation history:** Chat maintains context across multiple messages within a session
- **Quick action buttons:** Pre-set question shortcuts: "How do I negotiate this offer?", "Review my resume summary", "What should I include for a tech role?"
- **Citation of resume content:** Chat references specific sections of the user's resume when relevant
- **Pro tier gate:** Available on Pro and Premium tiers with usage limits; Free tier gets 5 free messages per day

**Out-of-Scope:**
- Voice / speech-to-text chat input
- Multi-modal input (upload images, documents)
- Real-time web search integration (accessing live job data)
- Personalized job recommendations
- Long-term memory across sessions (future)
- Email/draft generation via chat

---

### 4. User Stories

**US-P3.10-A:** As a **logged-in user on Pro/Premium**, I want to **type a career question in natural language** and get an **instant, intelligent answer**.

**US-P3.10-B:** As a **logged-in user on Pro/Premium**, I want the AI to **reference my resume data** when answering questions, so that I get **personalized advice**.

**US-P3.10-C:** As a **logged-in user**, I want to **use quick action buttons** for common questions, so that I can **get started without typing**.

**US-P3.10-D:** As a **logged-in user**, I want to **see my chat history** within a session, so that I can **reference previous advice**.

### 9. Dependencies & Assumptions

**Assumptions:**

### 10. Technical Notes

- **API Endpoint:** `POST /api/chat/` — accepts `{message: string, conversation_history: [{role, content}]}`. Returns `{reply: string, sources: [str]}`.
- **System Prompt:** "You are an AI Career Copilot. You help users with resume writing, job searching, interview preparation, salary negotiation, and career development. You have access to the user's resume data. Give specific, actionable, and honest advice. Be encouraging but realistic. Do not make up facts about the job market."
- **Resume Context:** Attach the user's current resume as context in the system prompt. Format: "User's Resume:\n[resume_data]\n\nUser's Question: [question]"
- **Conversation History:** Store up to 10 messages (5 user, 5 AI) in the request payload. Do not store history server-side.
- **Rate Limiting:** Track message count per user per day in Redis. Pro: 50/day, Premium: 100/day, Free: 5/day.
- **UI:** Floating chat button (bottom-right). Clicking opens a slide-in panel. Built with vanilla JS and CSS — no framework needed.
- **Streaming:** Use Server-Sent Events (SSE) for streaming responses. Show typing indicator while AI is generating.
- **Safety:** Add server-side content filter. Block requests containing PII or harmful content before sending to API.
